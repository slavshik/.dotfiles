# Git Hooks for Dotfiles — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add native git hooks (pre-commit lint/secret-scan) and a `dotup` pull-and-relink helper to this dotfiles repo, with zero npm/husky.

**Architecture:** Hooks live in a tracked `git-hooks/` dir activated by one `core.hooksPath` line in `install.sh`. A shared `_lib.sh` provides helpers; `pre-commit` checks only staged files and skips any missing tool with a warning (never blocks). `dotup` (a sourced zsh function) wraps `git pull --rebase` and re-runs `install.sh` only when it changed.

**Tech Stack:** POSIX/bash shell (target **bash 3.2** — macOS system bash), zsh functions, git `core.hooksPath`, shellcheck/shfmt/gitleaks/stylua/luacheck (all optional at runtime).

## Global Constraints

- **Target bash 3.2.57** (macOS `/bin/bash`). **No `mapfile`/`readarray`** — build arrays with `while IFS= read -r x; do arr+=("$x"); done < <(...)`. Do **not** use `set -u` (bash 3.2 errors on empty-array expansion).
- **Missing tools must never block a commit** — print a yellow skip warning and continue. Only real findings (lint error, format diff, detected secret) fail the hook.
- **Hook scripts:** shebang `#!/usr/bin/env bash`. The `pre-commit` entrypoint must be executable (`chmod +x`).
- **Conventional commits:** `type(scope): description`. End every commit message with `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- **`core.hooksPath` value is the relative path `git-hooks`** (portable, repo-local).
- Spec: `docs/superpowers/specs/2026-06-30-git-hooks-design.md`.

### Task 1: Shared hook helpers (`git-hooks/_lib.sh`)

**Files:**
- Create: `git-hooks/_lib.sh`

**Interfaces:**
- Consumes: nothing.
- Produces (for Task 2):
  - `staged <glob>...` → prints staged (A/C/M) file paths matching any glob, one per line, each file at most once.
  - `has <tool>` → exit 0 if `tool` is on PATH.
  - `skip <tool> [brew-hint]` → prints yellow `⚠ <tool> not installed — skipping (brew install <hint>)` to stderr; returns 0.
  - `fail <msg>` → prints red `✘ <msg>` to stderr and sets `HOOK_FAILED=1`.
  - `pass <msg>` → prints green `✔ <msg>`.
  - Global integer `HOOK_FAILED` (starts at `0`).

- [ ] **Step 1: Create `git-hooks/_lib.sh`**

```bash
#!/usr/bin/env bash
# Shared helpers for git hooks (sourced, not executed). Targets bash 3.2.

if [ -t 2 ]; then
  _C_YEL=$'\033[33m'; _C_RED=$'\033[31m'; _C_GRN=$'\033[32m'; _C_RST=$'\033[0m'
else
  _C_YEL=''; _C_RED=''; _C_GRN=''; _C_RST=''
fi

HOOK_FAILED=0

# staged <glob>...  → staged (added/copied/modified) files matching any glob.
staged() {
  local f g
  while IFS= read -r f; do
    for g in "$@"; do
      # shellcheck disable=SC2053  # glob match is intentional
      if [[ $f == $g ]]; then printf '%s\n' "$f"; break; fi
    done
  done < <(git diff --cached --name-only --diff-filter=ACM)
}

has() { command -v "$1" >/dev/null 2>&1; }

skip() {
  printf '%s⚠ %s not installed — skipping (brew install %s)%s\n' \
    "$_C_YEL" "$1" "${2:-$1}" "$_C_RST" >&2
}

fail() {
  printf '%s✘ %s%s\n' "$_C_RED" "$1" "$_C_RST" >&2
  HOOK_FAILED=1
}

pass() { printf '%s✔ %s%s\n' "$_C_GRN" "$1" "$_C_RST"; }
```

### Task 2: Pre-commit hook (`git-hooks/pre-commit`)

**Files:**
- Create: `git-hooks/pre-commit`

**Interfaces:**
- Consumes (from Task 1): `staged`, `has`, `skip`, `fail`, `pass`, `HOOK_FAILED`.
- Produces: an executable `pre-commit` hook that exits `0` on success / skips, `1` on any real finding.

**Notes:** Not activated yet (Task 5 sets `core.hooksPath`). Tests here run the hook **directly** against staged temp files, then unstage them.

- [ ] **Step 1: Create `git-hooks/pre-commit`**

```bash
#!/usr/bin/env bash
# Pre-commit: lint/format staged shell & lua, scan staged changes for secrets.
# Missing tools skip with a warning and never block the commit. Targets bash 3.2.
set -o pipefail

DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
# shellcheck source=_lib.sh
. "$DIR/_lib.sh"

# ── Shell: *.sh → shellcheck + shfmt ─────────────────────────────────────────
SH_FILES=(); while IFS= read -r f; do SH_FILES+=("$f"); done < <(staged '*.sh')
if [ ${#SH_FILES[@]} -gt 0 ]; then
  if has shellcheck; then
    if shellcheck "${SH_FILES[@]}"; then pass "shellcheck"; else fail "shellcheck found issues"; fi
  else skip shellcheck; fi
  if has shfmt; then
    if shfmt -d "${SH_FILES[@]}"; then pass "shfmt"; else fail "shfmt: formatting differs (run: shfmt -w <files>)"; fi
  else skip shfmt; fi
fi

# ── Zsh: *.zsh / zshrc → zsh -n (zsh is always installed) ────────────────────
ZSH_FILES=(); while IFS= read -r f; do ZSH_FILES+=("$f"); done < <(staged '*.zsh' '*zshrc*')
if [ ${#ZSH_FILES[@]} -gt 0 ]; then
  for f in "${ZSH_FILES[@]}"; do
    [ "$f" = ".fzf.zsh" ] && continue           # generated by fzf installer
    if zsh -n "$f" 2>/dev/null; then pass "zsh -n $f"; else zsh -n "$f"; fail "zsh syntax error: $f"; fi
  done
fi

# ── Lua: *.lua → stylua --check + luacheck ───────────────────────────────────
LUA_FILES=(); while IFS= read -r f; do LUA_FILES+=("$f"); done < <(staged '*.lua')
if [ ${#LUA_FILES[@]} -gt 0 ]; then
  if has stylua; then
    if stylua --check "${LUA_FILES[@]}"; then pass "stylua"; else fail "stylua: formatting differs (run: stylua <files>)"; fi
  else skip stylua; fi
  if has luacheck; then
    if luacheck "${LUA_FILES[@]}"; then pass "luacheck"; else fail "luacheck found issues"; fi
  else skip luacheck; fi
fi

# ── Secrets: gitleaks (staged-scan subcommand renamed across versions) ───────
if has gitleaks; then
  # `git` (gitleaks >=8.19) vs `protect` (older). Detect once, then run the real
  # scan WITHOUT suppressing stderr so a blocked secret shows its file/line/rule.
  if gitleaks git --help >/dev/null 2>&1; then _gl=git; else _gl=protect; fi
  if gitleaks "$_gl" --staged --redact; then pass "gitleaks"; else fail "gitleaks: potential secret detected"; fi
else
  skip gitleaks
fi

exit "$HOOK_FAILED"
```

### Task 3: `dotup` pull-and-relink helper (`zsh/scripts/dotfiles.zsh`)

**Files:**
- Create: `zsh/scripts/dotfiles.zsh`

**Interfaces:**
- Consumes: nothing (auto-sourced by `zsh/zshrc` via `for f in ~/.dotfiles/zsh/scripts/*.(sh|zsh)(N)`).
- Produces: shell function `dotup [dir]` (default `dir=$HOME/.dotfiles`).

- [ ] **Step 1: Create `zsh/scripts/dotfiles.zsh`**

```zsh
# dotup — update dotfiles, then re-run install.sh only if it changed.
# Reliable across merge/rebase/fast-forward (this repo has pull.rebase=true,
# so post-merge/post-rewrite hooks would not fire on a fast-forward pull).
dotup() {
  local dir="${1:-$HOME/.dotfiles}" before after changed
  before=$(git -C "$dir" rev-parse HEAD) || return
  git -C "$dir" pull --rebase || return
  after=$(git -C "$dir" rev-parse HEAD)
  [[ "$before" == "$after" ]] && return                  # nothing pulled
  changed=$(git -C "$dir" diff --name-only "$before" "$after")
  if grep -qx 'install.sh' <<< "$changed"; then
    echo "↻ install.sh changed — re-running"
    (cd "$dir" && ./install.sh)
  fi
  if grep -qx 'Brewfile' <<< "$changed"; then
    echo "⚠ Brewfile changed — review new packages"
  fi
}
```

### Task 5: Activate hooks via `install.sh` (go live)

**Files:**
- Modify: `install.sh` (insert a "Git hooks:" section before the "Git clones:" section)

**Interfaces:**
- Consumes: existing `ok`/`fail` helpers and `$DOTFILES` (both defined near the top of `install.sh`); the `git-hooks/pre-commit` file from Task 2.
- Produces: repo-local `core.hooksPath=git-hooks`, making the pre-commit hook live.

**Notes:** This is the only task whose own commit runs the hook (dogfood). Escape hatch if ever needed: `git commit --no-verify`.

- [ ] **Step 1: Insert the Git hooks section**

In `install.sh`, find:
```bash
echo ""
echo "Git clones:"
```
Insert immediately **before** those two lines:
```bash
echo ""
echo "Git hooks:"
if git -C "$DOTFILES" config core.hooksPath git-hooks; then
    ok "core.hooksPath → git-hooks"
else
    fail "core.hooksPath"
fi
chmod +x "$DOTFILES/git-hooks/pre-commit"
```

(Uses `if/then/else` rather than `cmd && ok || fail` to match install.sh's
style and avoid the SC2015 pattern — otherwise, once shellcheck is installed,
editing install.sh would make the hook block on its own activation line.)
